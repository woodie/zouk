# zouk v1.13.0

## Real PDF thumbnails resized to match the placeholder icon

A cached scan's real PDF thumbnail (`Image(nsImage: image)`) was rendered directly into the full 96x120 outer cell frame, filling it edge to edge -- noticeably larger than `DogEaredDocumentIcon`, the placeholder shown before a thumbnail loads (drawn at an inner 76x96, then centered in that same 96x120 outer frame). The real thumbnail now gets its own inner frame (82x104, tuned against real side-by-side comparisons) before being centered in the same outer frame, so a cell's visual size no longer jumps between "loading" and "loaded".

No other user-facing changes.
